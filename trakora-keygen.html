Keys validate against the same formula built into your Trakora app. Keep this file private — anyone with it can mint keys.
Privacy: the cloud stores only a one-way hash of each buyer's key+email — no plaintext name or email, so even a readable record leaks no PII. Buyer details stay only in this tool's local ledger on your machine.

① Simple rules — paste these to start issuing right away (Firebase → Realtime Database → Rules → Publish): { "rules": { "licenses": { "$key": { ".read": true, ".write": true } } } }
This lets you (and buyers' device activation) read & write the licenses path. It's the quickest way to get selling. Trade-off: in theory someone could write their own license record (self-issue) — but no PII is exposed (it's all hashed), so the only risk is piracy, which is inherent to any client-side license. Fine for getting started.

② Hardened rules — advanced, needs a Firebase admin secret: { "rules": { "licenses": { ".read": false, ".write": false, "$key": { ".read": true, "devices": { ".write": true, "$device": { ".validate": "newData.hasChild('at') && newData.child('at').isNumber() && (!newData.hasChild('ua') || (newData.child('ua').isString() && newData.child('ua').val().length <= 200))" } } } } } }
These deny key-creation to the public (stops self-issued keys) — so you must add your Firebase Database admin secret in Advanced settings to issue (it bypasses the rules). Note: newer Firebase projects have database secrets disabled by default; if you don't have one, use the Simple rules above. After pasting either set, click “Test connection”.
